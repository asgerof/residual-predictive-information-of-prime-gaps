# Residual Predictive Information in Prime Gaps: A Prequential Falsification Protocol and a Negative Result Beyond Wheel-First-Hit Baselines

**Author:** [Name]  
**Date:** June 28, 2026  
**Status:** First manuscript draft

## Abstract

Prime gaps are deterministic, but many questions about their statistical structure are naturally phrased using probabilistic models. This paper studies a relative version of such a question: after known arithmetic structure has been built into an explicit online null model, does previous gap history still improve prediction of the next prime gap?

For dyadic windows \([X,2X)\), let \(G_X\) denote the block of consecutive prime gaps beginning at primes \(p_n \in [X,2X)\). We define residual predictive information as a prequential code gain
\[
R_k(X)=\frac{L_{B_k}(G_X)-L_{U_k}(G_X)}{|G_X|},
\]
where \(B_k\) is an arithmetic baseline and \(U_k\) is a penalized online residual predictor using the same permitted arithmetic information together with past gaps. Positive \(R_k(X)\) means that the residual learner compresses the observed gap sequence better than the arithmetic baseline after paying its own online model cost.

The implemented paper-scale suite evaluates this protocol through \(X=2^{26}\). The residual machinery detects strong missing structure under a weak local-density baseline \(B_0\): a rank-mod-64 context-tree residual coder obtains \(R=0.2674244536300398\) bits per gap, corresponding to approximately \(974{,}961\) bits of total code gain at \(X=2^{26}\). However, the same machinery gives null-level behavior once the wheel-first-hit baseline \(B_1(11)\) is used. The main \(B_1(11)\) bucket-residual test gives
\[
R=-2.7429243523407067\cdot 10^{-7}
\]
bits per gap at \(X=2^{26}\), with \(z=-0.318\) against 200 stop-time synthetic null replicates. The corresponding null 5--95% interval is
\[
[-2.7445474076652\cdot 10^{-7},-2.7406420776259464\cdot 10^{-7}].
\]

Thus, under the tested residual predictors and through \(X=2^{26}\), no positive residual predictive information is detected beyond the \(B_1(11)\) wheel-first-hit baseline. The contribution is a reproducible prequential falsification protocol and a finite-scale negative result, not an asymptotic theorem and not a proof that prime gaps contain no residual information.

## 1. Introduction

The sequence of prime numbers is deterministic. In an absolute sense, therefore, the primes are not random and the first \(N\) primes can be generated by a short algorithm. Nevertheless, probabilistic models of primes are central in analytic number theory. Cramer-type models, random-sieve models, and Hardy--Littlewood tuple heuristics all treat the primes as if they were random objects subject to arithmetic constraints. These models do not assert literal randomness. They provide a language for distinguishing expected structure from unexplained structure.

This paper asks a deliberately relative question. It does not ask whether prime gaps are compressible in an absolute sense, nor whether they are random. It asks:

> After specified arithmetic structure has been built into an explicit online null model, does previous prime-gap history improve prediction of the next gap?

The distinction is important. A residual model should not receive credit for rediscovering elementary arithmetic structure that was omitted from the baseline. For example, a baseline that models only local prime density but ignores small-prime congruence constraints is too weak. A flexible residual learner can detect the missing wheel structure and produce a large apparent code gain. Such a gain is real relative to the weak baseline, but it is not evidence for residual sequential information beyond known arithmetic constraints.

The protocol developed here measures residual predictive information using prequential code length. A baseline \(B_k\) assigns an online probability to each next gap using declared arithmetic information. A residual predictor \(U_k\) is allowed to use past gap history, but it must be online and penalized. The test statistic is the per-gap code gain
\[
R_k(X)=\frac{L_{B_k}(G_X)-L_{U_k}(G_X)}{|G_X|}.
\]
Positive values indicate that past gap history improves prediction beyond the baseline; null-level or negative values indicate no detected residual gain after the complexity cost of the residual learner.

The main empirical result is negative. The implemented residual models detect strong missing structure under a weak \(B_0\) baseline, but the signal disappears under the wheel-first-hit baseline \(B_1(11)\). The result should be read as a finite-scale falsification result: at the tested scale and for the tested residual predictors, apparent residual predictability does not survive a stronger arithmetic null.

The paper makes four contributions.

1. It formulates a prequential information-theoretic protocol for testing residual predictive information in prime gaps.
2. It introduces an explicit arithmetic-baseline hierarchy, separating weak local-density models from wheel-corrected and higher-order arithmetic baselines.
3. It demonstrates that the residual machinery is active by showing a strong positive control against \(B_0\).
4. It reports a finite-scale negative result beyond \(B_1(11)\): no positive residual predictive information is detected through \(X=2^{26}\) under the tested residual coders.

The claim is intentionally narrow. The paper does not claim that prime gaps contain no residual information, does not claim that all possible higher-order arithmetic baselines have been exhausted, and does not prove an asymptotic theorem.

## 2. Observation Protocol

Let
\[
p_1 < p_2 < p_3 < \cdots
\]
be the increasing sequence of primes, and define the consecutive prime gaps
\[
g_n=p_{n+1}-p_n.
\]

For a scale parameter \(X\), define the dyadic index set
\[
I_X=\{n:X\leq p_n<2X\},
\]
and the corresponding gap block
\[
G_X=(g_n)_{n\in I_X}.
\]

All prediction is prequential. Before \(g_n\) is revealed, a predictor must assign a probability distribution to the next gap using only information available at that time. For a predictor \(Q\), write
\[
Q_n(\cdot\mid \mathcal F_{n,k})
\]
for the distribution assigned to \(g_n\), where \(\mathcal F_{n,k}\) denotes the information field allowed at baseline level \(k\).

The prequential code length of \(G_X\) under \(Q\) is
\[
L_Q(G_X)=\sum_{n\in I_X} -\log_2 Q_n(g_n\mid \mathcal F_{n,k}).
\]
The mean code length per gap is
\[
h_Q(X)=\frac{L_Q(G_X)}{|I_X|}.
\]

The central statistic is the baseline-relative code gain
\[
R_k(X)=\frac{L_{B_k}(G_X)-L_{U_k}(G_X)}{|I_X|}.
\]

Here \(B_k\) is an explicit arithmetic baseline and \(U_k\) is a penalized online residual predictor. The residual predictor may use previous gaps, but it may not use future gaps, primality testing of candidate offsets in the test window, or any hidden reconstruction of the target prime sequence.

The interpretation is:

- \(R_k(X)>0\): the residual predictor improves on the arithmetic baseline by \(R_k(X)\) bits per gap.
- \(R_k(X)\approx 0\): no residual predictive information is detected at the tested scale and model resolution.
- \(R_k(X)<0\): the residual predictor performs worse after model complexity is paid.

This is a relative information measure. It is not an entropy of the primes and not a randomness test in the absolute sense.

## 3. Arithmetic Null Models

The main methodological requirement is that a residual predictor should be compared against a fair arithmetic baseline. If the baseline omits simple arithmetic constraints, then a residual learner may obtain a positive code gain merely by rediscovering those constraints.

We use a nested hierarchy
\[
B_0\subset B_1\subset B_2\subset B_3\subset\cdots
\]
of progressively stronger arithmetic baselines.

### 3.1 Local-density baseline \(B_0\)

The weakest baseline \(B_0\) uses only the local density of primes near \(p_n\). Since primes near \(x\) have density approximately \(1/\log x\), a simple even-gap model takes
\[
B_0(g_n=2j\mid p_n)=(1-\theta_n)^{j-1}\theta_n,
\qquad
\theta_n\approx \frac{2}{\log p_n}.
\]

This baseline is deliberately weak. It accounts for the local scale of gaps but does not account for small-prime divisibility constraints beyond parity. As a result, it should be expected to leave detectable arithmetic structure in the residual sequence.

### 3.2 Wheel-first-hit baseline \(B_1(y)\)

The wheel-corrected baseline \(B_1(y)\) incorporates congruence constraints modulo
\[
W_y=\prod_{q\leq y}q.
\]

Conditional on \(p_n\bmod W_y\), many offsets \(m\) are impossible because
\[
\gcd(p_n+m,W_y)>1.
\]
The wheel baseline places probability only on offsets that survive these finite congruence obstructions.

The implementation used for the headline result is \(B_1(11)\), for which
\[
W=2\cdot3\cdot5\cdot7\cdot11=2310,
\qquad
\phi(W)=480.
\]

For each residue class of \(p_n\bmod W\), the baseline orders the admissible offsets and uses a first-hit model on this ordered set. The hit probability is calibrated to match the local prime density. This baseline is still simple, but it absorbs a large amount of small-modulus arithmetic structure that a \(B_0\) residual learner can otherwise rediscover.

### 3.3 Higher-order baselines

A natural \(B_2\) baseline would incorporate finite tuple corrections. For a finite offset set
\[
\mathcal H=\{0,h_1,\ldots,h_s\},
\]
Hardy--Littlewood heuristics predict correction by the singular series
\[
\mathfrak S(\mathcal H)=
\prod_q
\left(1-\frac1q\right)^{-|\mathcal H|}
\left(1-\frac{\nu_q(\mathcal H)}q\right),
\]
where \(\nu_q(\mathcal H)\) is the number of occupied residue classes modulo \(q\).

The repository contains several \(B_2\)-style prototype diagnostics, including pair singular-series reweighting, residue-transition calibration, finite consecutive-prime inclusion-exclusion, and endpoint/exclusion shrinkage. These are not treated as a canonical or exhaustive tuple-corrected hierarchy. They are useful diagnostics but not a basis for claiming that all \(B_2\)-level arithmetic structure has been ruled out.

A still stronger \(B_3\)-style baseline could incorporate global analytic corrections, such as predetermined smooth corrections from \(\operatorname{Li}(x)\), explicit-formula terms using a bounded set of zeta zeros, or related global intensity corrections. No such \(B_3\)-style correction is implemented in the present paper-scale suite.

## 4. Residual Predictors

For each baseline \(B_k\), the residual predictor \(U_k\) is constructed so that it modifies only a declared residual symbolization while preserving the baseline's within-symbol distribution. This prevents the residual model from becoming a hidden replacement for the arithmetic baseline.

The general construction is:
\[
U(g\mid \text{history},p)
=
U_s(s(g,p)\mid \text{history},p)\,
B(g\mid p,s(g,p)),
\]
where \(s(g,p)\) is a residual symbol associated with the gap and current prime, \(U_s\) is an online predictor for the symbol sequence, and \(B(g\mid p,s(g,p))\) is the baseline distribution conditional on the symbol.

Two residual families are used in the paper-scale suite.

### 4.1 Bucket residual model

The main \(B_1(11)\) test uses buckets of the normalized gap
\[
\frac{g_n}{\log p_n}.
\]
The residual model is a Bayesian two-expert mixture between:

1. the baseline bucket distribution induced by \(B_1(11)\);
2. a Krichevsky--Trofimov-smoothed finite-depth Markov predictor on previous buckets.

The mixture includes the baseline expert. Therefore, a positive residual gain requires the sequential bucket expert to improve prediction enough to overcome its own mixture cost.

### 4.2 Rank-mod-64 CTW residual model

The main positive-control residual model uses a rank symbol. Under the wheel-first-hit model, each admissible offset has a first-hit rank. The symbol is the admissible first-hit rank modulo 64:
\[
s(g,p)=\operatorname{rank}_{B_1}(g,p)\bmod 64.
\]

A sparse context-tree weighting style predictor is then applied to the symbol sequence. The same residual machinery is evaluated against both \(B_0\) and \(B_1(11)\). This is important: a method that detects structure under \(B_0\) but not under \(B_1(11)\) supports the interpretation that the detected \(B_0\) signal is omitted wheel structure rather than robust residual sequential information.

## 5. Synthetic Nulls and Positive-Claim Criteria

For each baseline, synthetic null sequences are generated from the same baseline and evaluated using the same residual machinery. The headline paper-scale runs use stop-time nulls: for each dyadic window, the synthetic process continues until it reaches the same endpoint scale rather than being forced to match the exact number of real gaps.

Stop-time nulls are preferable for the present question because the number of gaps in a window is itself part of the stochastic behavior induced by the baseline. Fixed-count nulls can be useful diagnostics, but the main significance comparison should not condition away endpoint behavior unless that conditioning is explicitly intended.

A positive residual-information claim would require all of the following:

1. positive code gain beyond \(B_1(11)\);
2. survival against stop-time synthetic nulls;
3. agreement across at least two independent residual symbolizations;
4. absence, or substantial weakening, of the same effect in synthetic null sequences;
5. persistence over increasing dyadic windows;
6. chronological tuning only.

The completed paper-scale suite does not satisfy these criteria. The positive-control signal is strong under \(B_0\), but it disappears under \(B_1(11)\).

## 6. Experiments

The paper-scale experiments use dyadic windows with
\[
X=2^m
\]
and test exponents through \(m=26\). The main runs use 200 synthetic null replicates/checkpoints and a common seed.

The completed paper-scale suite consists of three experiment blocks.

### 6.1 Main \(B_1(11)\) bucket-residual test

This is the primary residual-information test beyond the wheel-first-hit baseline. It uses:

- baseline: \(B_1(11)\);
- residual symbolization: buckets of \(g/\log p\);
- residual learner: finite-depth Markov bucket predictor in a baseline/residual mixture;
- depth: 4;
- null mode: stop-time;
- null replicates: 200.

### 6.2 Rank-mod-64 CTW control

This test uses the rank-mod-64 residual symbolization with a sparse context-tree residual learner. It is evaluated under both \(B_0\) and \(B_1(11)\). Its purpose is to show whether the residual machinery detects missing arithmetic structure under a deliberately weak baseline, and whether the same effect remains after wheel-first-hit structure is included.

The configuration is:

- residual symbolization: rank modulo 64;
- residual learner: sparse CTW-style context mixture;
- depth: 1;
- baselines: \(B_0\) and \(B_1(11)\);
- null mode: stop-time;
- null replicates: 200.

### 6.3 Exact B0/B1 baseline ladder

The baseline ladder compares exact baseline code lengths under \(B_0\) and \(B_1(11)\). This quantifies how much arithmetic structure is absorbed by the wheel-first-hit baseline relative to the local-density baseline.

## 7. Results

### 7.1 Main \(B_1(11)\) bucket-residual result

At \(X=2^{26}=67{,}108{,}864\), the main \(B_1(11)\) bucket-residual test contains
\[
n=3{,}645{,}744
\]
gaps.

The observed residual code gain is
\[
R=-2.7429243523407067\cdot10^{-7}
\]
bits per gap.

The stop-time synthetic null distribution has mean
\[
-2.7425414055848714\cdot10^{-7},
\]
with 5--95% interval
\[
[-2.7445474076652\cdot10^{-7},
-2.7406420776259464\cdot10^{-7}].
\]
The observed value has
\[
z=-0.31818465586295164
\]
against this null distribution.

This is ordinary null-level behavior. It gives no evidence for positive residual predictive information beyond \(B_1(11)\).

### 7.2 Positive control against \(B_0\)

The rank-mod-64 CTW residual model gives a strong positive result against the weak \(B_0\) baseline. At \(X=2^{26}\),
\[
R=0.2674244536300398
\]
bits per gap, corresponding to total code gain
\[
974{,}961.0972749958
\]
bits. The empirical upper-tail p-value is
\[
p_{\geq}=0.004975124378109453.
\]

This confirms that the residual machinery is active. It can detect missing structure when the arithmetic baseline is too weak.

### 7.3 Same CTW control under \(B_1(11)\)

Under \(B_1(11)\), the same rank-mod-64 CTW residual model gives
\[
R=-2.742924352340597\cdot10^{-7}
\]
bits per gap, corresponding to approximately \(-1\) bit of total gain. The empirical upper-tail p-value is
\[
p_{\geq}=0.6119402985074627.
\]

Thus, the same residual machinery that strongly separates from the \(B_0\) null gives null-level behavior under \(B_1(11)\). This is the central empirical pattern of the paper.

### 7.4 Exact baseline ladder

At \(X=2^{26}\), the exact baseline code lengths are:

\[
B_0 = 4.563439906212204
\]
bits per gap, and
\[
B_1(11)=3.1704284777012335
\]
bits per gap.

The improvement of \(B_1(11)\) over \(B_0\) is therefore
\[
1.3930114285109703
\]
bits per gap.

This is large compared with the residual effects being tested after \(B_1(11)\). The result explains why a weak baseline can generate apparent residual structure: much of that structure is simply small-modulus arithmetic information not yet included in the null.

### 7.5 Cross-exponent robustness

The negative \(B_1(11)\) result is not only an endpoint observation. Across the tested dyadic exponents, the main \(B_1(11)\) bucket residual remains inside the 5--95% stop-time null band and has small absolute z-scores. The rank-mod-64 CTW positive control separates from \(B_0\) across tested exponents, while the same residual model remains null-consistent under \(B_1(11)\). The baseline ladder also keeps \(B_1(11)\) below \(B_0\) across the tested range.

These checks support the qualitative conclusion: the protocol detects missing structure under weak baselines, but does not detect positive residual predictive information beyond the implemented wheel-first-hit baseline.

## 8. Discussion

The main lesson is methodological. Patterns in prime gaps are easy to find, but a pattern should not be counted as residual predictive information unless it survives an explicit arithmetic null model and a penalized prequential prediction test.

The \(B_0\) result shows that residual learners can produce large apparent gains when the baseline omits known arithmetic structure. The \(B_1(11)\) result shows that this gain disappears after small-modulus wheel structure is included. This is evidence that the detected \(B_0\) signal is arithmetic baseline structure, not robust residual sequential information.

This result is negative, but it is not empty. A useful falsification protocol should be able to kill its own positive hypothesis. Here, the same machinery that finds a large signal under a weak baseline fails to find one under a stronger baseline. That is precisely the kind of behavior expected from a well-controlled residual-information test.

The result also clarifies the role of higher-order baselines. A stronger \(B_2\) or \(B_3\) model may absorb additional structure, but the present paper does not need such a model to make its main negative claim. The claim is limited to the tested residual predictors beyond \(B_1(11)\). Future work can strengthen the hierarchy, but the current empirical conclusion is already informative: the first level at which small-prime wheel structure is properly included is sufficient to eliminate the tested residual signals through \(X=2^{26}\).

## 9. Limitations

The limitations are substantial and should be stated plainly.

First, the result is finite-scale. The computations reach \(X=2^{26}\), not an asymptotic regime. Nothing here proves that residual predictive information vanishes as \(X\to\infty\).

Second, the residual model classes are limited. The paper tests bucket-based Markov residuals and rank-mod-64 CTW-style residuals. Other symbolizations, deeper context models, continuous predictors, neural sequence models, or analytically motivated residual models could behave differently. A positive result from such a model would need to pass the same prequential and synthetic-null criteria.

Third, \(B_1(11)\) is not a complete arithmetic model of prime gaps. It incorporates finite wheel constraints up to 11, but it does not incorporate a canonical tuple-corrected \(B_2\) hierarchy or global analytic corrections.

Fourth, the implemented \(B_2\)-style diagnostics are not exhaustive. They should be treated only as auxiliary checks.

Fifth, the number of synthetic null replicates is 200. This is adequate for a first finite-scale negative study and for detecting gross departures from the null, but it is not designed to support extremely small p-value claims.

These limitations define the scope of the paper. The contribution is a protocol and a controlled finite-scale result, not a final theory of prime-gap predictability.

## 10. Conclusion

This paper introduced a prequential residual-coding protocol for measuring predictive information in consecutive prime gaps relative to explicit arithmetic null models. The central statistic is the per-gap code gain
\[
R_k(X)=\frac{L_{B_k}(G_X)-L_{U_k}(G_X)}{|G_X|}.
\]

The protocol distinguishes residual predictability from omitted baseline structure. Against a weak \(B_0\) baseline, the rank-mod-64 CTW residual model obtains a large positive code gain. Against the wheel-first-hit baseline \(B_1(11)\), the same residual machinery gives null-level behavior. The main \(B_1(11)\) bucket-residual test also gives null-level behavior through \(X=2^{26}\).

The safest conclusion is therefore finite-scale and model-class-specific:

> Under the tested online residual predictors and through \(X=2^{26}\), no positive residual predictive information is detected beyond the \(B_1(11)\) wheel-first-hit baseline.

This does not prove that prime gaps contain no residual information. It shows that a controlled residual-coding protocol can separate weak-baseline artifacts from robust residual signals, and that the tested positive signal disappears once small-prime wheel structure is included.
