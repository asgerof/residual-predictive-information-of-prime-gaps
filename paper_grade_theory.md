# Residual Predictive Information in Prime Gaps

Version: paper-facing theory note, June 28 2026.

## Abstract

This note formulates a testable information-theoretic protocol for consecutive
prime gaps. The aim is not to ask whether the primes are compressible in an
absolute sense: they are generated by a short deterministic algorithm. The aim
is instead to ask whether the sequence of consecutive prime gaps contains
measurable sequential predictive information after known arithmetic structure
has already been built into an explicit null model.

For dyadic windows \([X,2X)\), we compare the prequential log-loss of an
arithmetic baseline \(B_k\) with that of a penalized online residual predictor
\(U_k\). The resulting statistic

\[
R_k(X)=\frac{L_{B_k}(G_X)-L_{U_k}(G_X)}{|G_X|}
\]

is the per-gap code gain, in bits, obtained by learning additional sequential
structure beyond the arithmetic information already supplied to the baseline.
Positive values indicate residual predictability; null-level or negative values
indicate that the residual learner has not improved on the arithmetic baseline
after its own complexity cost.

The accompanying paper-scale experiments reach \(X=2^{26}\) with 200
stop-time synthetic null replicates/checkpoints for the main tests. The protocol
detects strong missing structure against the weak \(B_0\) baseline, but no
positive residual predictive information is detected beyond the wheel-first-hit
baseline \(B_1(11)\). The present contribution should therefore be framed as a
reproducible falsification protocol and finite-scale negative result, not as a
proof that residual information is absent.

## 1. Motivation

Let \(p_1<p_2<\cdots\) denote the primes and let

\[
g_n=p_{n+1}-p_n
\]

be the sequence of consecutive prime gaps. Since the primes are computable,
ordinary absolute compressibility is not the right invariant. A program that
merely asks whether the first \(N\) prime gaps can be compressed confounds
determinism, lookup, primality testing, and arithmetic structure.

The question studied here is relative and predictive:

> After a specified amount of arithmetic structure has been built into a null
> model, does the previous gap history improve online prediction of the next
> prime gap by a positive number of bits per gap?

Equivalently, the object of study is not randomness of the primes in the
absolute sense, but residual sequential dependence relative to an explicit
arithmetic baseline.

This distinction is central for paper framing. A positive result would require
a residual signal that survives increasingly strong arithmetic nulls. The
current result is the opposite: apparent predictability under a weak baseline is
absorbed by a wheel-first-hit baseline.

## 2. Observation Protocol

For \(X\geq 2\), define the dyadic index set

\[
I_X=\{n:X\leq p_n<2X\},
\]

and the gap block

\[
G_X=(g_n)_{n\in I_X}.
\]

All prediction is prequential. At time \(n\), before \(g_n\) is revealed, a
predictor assigns a probability distribution

\[
Q_n(\cdot\mid \mathcal F_{n,k})
\]

to the next gap. Here \(\mathcal F_{n,k}\) is the information field allowed at
baseline level \(k\).

The prequential code length of \(G_X\) under \(Q\) is

\[
L_Q(G_X)
=\sum_{n\in I_X}-\log_2 Q_n(g_n\mid \mathcal F_{n,k}).
\]

The mean code length per gap is

\[
h_Q(X)=\frac{L_Q(G_X)}{|I_X|}.
\]

This is equivalent to cumulative logarithmic loss. It is important that the
predictor is evaluated only on predictions made before the corresponding gap is
observed.

## 3. Admissible Information

The central technical issue is to prevent the predictor from reconstructing the
prime sequence while still allowing it to use explicitly declared arithmetic
structure. We therefore distinguish between arithmetic features and forbidden
oracles.

At level \(k\), an admissible predictor may use:

1. previous gaps \(g_m\) for \(m<n\);
2. normalized previous gaps such as \(g_m/\log p_m\);
3. the current scale, including \(p_n\), \(n\), \(X\), and \(\log p_n\);
4. congruence information modulo a specified finite set of small primes;
5. wheel-admissible offsets relative to \(p_n\);
6. finite local tuple features of bounded order;
7. parameters estimated only from earlier windows \(G_Y\) with \(2Y\leq X\);
8. any deterministic computation applied to the preceding data and to the
   explicitly allowed feature maps.

An admissible predictor may not use:

1. primality testing of candidate points \(p_n+m\);
2. direct sieving by primes outside the allowed finite modulus set;
3. future gaps or future primes;
4. a table, hash, compressed representation, or other encoding of the actual
   target gap block;
5. a procedure whose effect is to reconstruct the primes in the test window.

For a fully formal treatment, admissibility can be defined by feature maps

\[
\Phi_k(n,X,p_n,g_1,\ldots,g_{n-1})
\]

and by requiring predictors at level \(k\) to be measurable with respect to the
sigma-field generated by \(\Phi_k\). This turns the informal "no hidden
primality oracle" condition into a precise modelling restriction.

## 4. Arithmetic Baseline Hierarchy

The null model is not a single universal object. It is a nested hierarchy

\[
B_0\subset B_1\subset B_2\subset B_3\subset\cdots
\]

of progressively stronger arithmetic baselines. Each \(B_k\) is itself an
online predictive distribution for the next gap.

### 4.1 Local Density Baseline \(B_0\)

The weakest baseline uses only the local prime density near \(p_n\). It models
the next admissible point as occurring with local hazard approximately

\[
\lambda_n\asymp \frac{1}{\log p_n}.
\]

After the exceptional prime 2, gaps are supported on even positive integers. A
simple version is the discrete geometric model

\[
B_0(g_n=2j\mid p_n)
=(1-\theta_n)^{j-1}\theta_n,
\qquad \theta_n\approx \frac{2}{\log p_n}.
\]

More refined versions may use standard local corrections to match the expected
gap at scale \(p_n\). The paper-scale experiments show that \(B_0\) is too weak
for the residual question: residual coders easily recover missing wheel
structure from this baseline.

### 4.2 Wheel-Corrected Baseline \(B_1(y)\)

Let

\[
W_y=\prod_{q\leq y}q
\]

be the primorial over primes up to \(y\). Conditional on the residue class of
\(p_n\) modulo \(W_y\), some offsets \(m\) are impossible because

\[
\gcd(p_n+m,W_y)>1.
\]

The wheel-corrected baseline places mass only on offsets \(m\) that survive this
finite congruence obstruction. The implementation used in the headline tests is
a first-hit model on the ordered set of \(W_y\)-admissible offsets, with hazard
calibrated so that the mean spacing agrees with the local prime density.

The completed paper-scale run uses \(B_1(11)\), where \(W=2310\) and
\(\phi(W)=480\). At the tested scale, this baseline absorbs the residual signal
seen under \(B_0\).

### 4.3 Tuple-Corrected Baseline \(B_2(y,r)\)

The wheel model corrects one-point admissibility but not the expected frequency
of local constellations. For a finite offset set

\[
\mathcal H=\{0,h_1,\ldots,h_s\},
\qquad s\leq r,
\]

Hardy-Littlewood heuristics predict a correction by the singular series

\[
\mathfrak S(\mathcal H)
=\prod_q
\left(1-\frac1q\right)^{-|\mathcal H|}
\left(1-\frac{\nu_q(\mathcal H)}q\right),
\]

where \(\nu_q(\mathcal H)\) is the number of occupied residue classes modulo
\(q\).

A canonical \(B_2\) baseline would incorporate finite tuple corrections for
patterns of order at most \(r\), truncated or approximated using primes
\(q\leq y\), without learning from the test block. Equivalently, it would be a
finite-order arithmetic model for local constellations with probabilities fixed
by explicit singular-series-type corrections.

The repository contains several \(B_2\)-style prototype attempts, including
pair singular-series reweighting, residue-transition calibration, finite
consecutive-prime inclusion-exclusion, and two-parameter endpoint/exclusion
shrinkage. These are diagnostics, not a canonical exhaustion of \(B_2\). The
paper should state only that these finite prototypes did not improve exact
prequential log-loss over \(B_1(11)\) at the tested scales. It should not claim
that all possible tuple-corrected arithmetic baselines have been ruled out.

### 4.4 Analytically Corrected Baseline \(B_3(y,r,z)\)

A stronger baseline may also incorporate global analytic information about
fluctuations in prime density. Examples include smooth approximations derived
from \(\operatorname{Li}(x)\), explicit-formula terms using a bounded set of
zeta zeros, or other predetermined corrections to the local intensity.

The parameter \(z\) controls the amount of global analytic information. This
level must be handled with particular care: if \(z\) is allowed to grow too
quickly relative to \(X\), analytic correction can become a disguised
reconstruction of prime counts in short intervals. A paper-grade formulation
should therefore impose an explicit growth rule for \(z=z(X)\), or treat \(z\)
as fixed before testing on \(G_X\).

No \(B_3\)-style global analytic correction is implemented in the current
paper-scale suite.

## 5. Residual Model Class and Universal Coding

For each baseline \(B_k\), let \(\mathcal A_k\) be a class of admissible online
predictors that may use the same arithmetic features as \(B_k\), together with
preceding gap history. The purpose of \(\mathcal A_k\) is to test for
additional sequential dependence not already captured by the arithmetic null.

To avoid overfitting, we do not compare \(B_k\) with the best model in
\(\mathcal A_k\) without penalty. Instead we use a universal or penalized
online code \(U_k\) over \(\mathcal A_k\). Examples include:

1. Bayesian mixtures over countable model families;
2. normalized maximum likelihood or sequential normalized maximum likelihood,
   when available;
3. context-tree weighting for finite discretizations of normalized gaps;
4. penalized Markov or autoregressive models with prequential complexity
   penalties;
5. online learning algorithms with regret bounds against \(\mathcal A_k\).

The defining requirement is a regret inequality of the form

\[
L_{U_k}(G_X)
\leq
\inf_{A\in\mathcal A_k}
\{L_A(G_X)+\operatorname{pen}_k(A,X)\}
+o(|I_X|),
\]

where \(\operatorname{pen}_k(A,X)\) is the cost of model complexity. Thus a
code-length improvement counts as evidence only if it survives its own
description cost.

## 6. Residual Predictive Information

The residual predictive information at level \(k\) is

\[
R_k(X)
=
\frac{L_{B_k}(G_X)-L_{U_k}(G_X)}{|I_X|}.
\]

Interpretation:

1. \(R_k(X)>0\): the universal residual model predicts the next gap better than
   the arithmetic baseline by \(R_k(X)\) bits per gap.
2. \(R_k(X)\approx 0\): no residual sequential information is detected beyond
   the baseline at the given scale and model resolution.
3. \(R_k(X)<0\): the residual model is worse after complexity penalty.

The quantity is a relative information measure. It is not an entropy of the
prime gaps in isolation, and it is not a claim that the primes are random.

## 7. Completed Paper-Scale Empirical Result

The completed paper-scale suite tests the main empirical question through
\(X=2^{26}\) with 200 null replicates/checkpoints:

> Does an online residual predictor detect positive residual predictive
> information in consecutive prime gaps after the wheel-first-hit baseline
> \(B_1(11)\) has already absorbed small-modulus arithmetic structure?

The result is negative at the tested scale.

At \(X=2^{26}\), the main \(B_1(11)\) bucket-residual run gives

\[
R=-2.7429243523407067\cdot 10^{-7}
\]

bits per gap with \(z=-0.318\) versus the stop-time synthetic null distribution.
The corresponding null mean is

\[
-2.7425414055848714\cdot 10^{-7},
\]

with a 5--95% interval

\[
[-2.7445474076652\cdot 10^{-7},
 -2.7406420776259464\cdot 10^{-7}].
\]

Thus the observed value is ordinary null-level behavior, not positive residual
code gain.

As a positive control, the rank-mod-64 CTW residual learner detects strong
missing structure against \(B_0\): at \(X=2^{26}\), \(R=0.2674244536300398\)
bits per gap and the total code gain is approximately \(974{,}961\) bits. The
same residual learner gives null-level behavior against \(B_1(11)\), with total
gain approximately \(-1\) bit and empirical \(p_{\geq}\approx 0.612\).

The exact baseline ladder quantifies why this matters. At \(X=2^{26}\),
\(B_1(11)\) improves over \(B_0\) by \(1.3930114285109703\) bits per gap. The
method therefore distinguishes a weak null from a stronger arithmetic null:
it is sensitive enough to find missing structure, but that structure is
absorbed by the wheel-first-hit model.

## 8. Asymptotic Questions

For fixed baseline level \(k\), define, when the limit exists,

\[
\rho_k=\lim_{X\to\infty}R_k(X).
\]

For a nested baseline hierarchy, define, again when the limit exists,

\[
\rho_\infty=\lim_{k\to\infty}\rho_k.
\]

The weak residual predictive information conjecture is:

> The limit \(\rho_\infty\) exists for a natural nested hierarchy of arithmetic
> baselines and admissible universal predictors.

The strong residual predictive information conjecture is:

> \(\rho_\infty>0\).

The current paper-scale experiments do not support the strong conjecture beyond
\(B_1(11)\) at \(X\leq 2^{26}\). They do not prove \(\rho_\infty=0\). The
appropriate conclusion is finite-scale, model-class-specific, and negative.

The null expectation suggested by Hardy-Littlewood and random-sieve heuristics
is that sufficiently strong arithmetic baselines should exhaust measurable
sequential predictability. In that sense, a well-controlled negative result is
not a failure of the protocol; it is the protocol doing its job.

## 9. Local Residual Resonance

For logarithmic windows

\[
[e^u,e^{u+\Delta}),
\]

define \(R_k(u,\Delta)\) analogously. A residual resonance zone is a region in
\((u,\Delta)\)-space where \(R_k(u,\Delta)\) is significantly larger than
expected under synthetic sequences generated from \(B_k\).

Such a zone is evidence only if it satisfies all of the following:

1. it survives local density correction;
2. it survives wheel correction;
3. it survives finite tuple/singular-series correction;
4. it survives global analytic correction;
5. it is stable under changes of window size;
6. it is stable under alternative codings and model classes;
7. it is absent, or substantially weaker, in synthetic null sequences;
8. it is validated out of sample on later windows.

Residual resonance should be treated as a diagnostic, not as a primary
invariant. The current paper should not be organized around resonance claims.

## 10. Falsification Criteria

The strong conjecture is weakened or falsified if:

1. \(R_k(X)\) tends to zero as \(k\) increases;
2. the effect disappears under alternative universal codes;
3. the effect is present only for one residual model class;
4. the same effect appears in synthetic null sequences from \(B_k\);
5. the effect is explained by small-modulus or primorial structure;
6. the gain fails to survive tuple/singular-series correction;
7. the gain fails to survive model-complexity penalty;
8. the gain fails in out-of-sample prediction on later windows;
9. the gain depends on analytic correction parameters that effectively encode
   the test window.

These criteria are part of the theory rather than an afterthought. The program
is designed to be capable of killing its own positive hypothesis. The completed
paper-scale suite does exactly that for the tested \(B_1(11)\) residual models.

## 11. Experimental Protocol

A minimal empirical study should proceed as follows.

1. Generate consecutive prime gaps in dyadic windows \([X,2X)\).
2. Split windows chronologically; train or tune only on earlier windows.
3. Implement \(B_0\), \(B_1(y)\), and clearly labeled finite \(B_2\)-style
   diagnostics.
4. Optionally implement \(B_3(y,r,z)\) with a fixed, predeclared analytic
   correction rule.
5. Choose one or more universal online residual models \(U_k\).
6. Record cumulative log-loss and code length per gap.
7. Compare \(R_k(X)\) with the distribution of code gains on synthetic
   sequences generated from the same baseline.
8. Test robustness over \(X\), \(y\), \(r\), window width, coding scheme, and
   residual model class.
9. Report both positive findings and vanishing results.

The key empirical question is not whether patterns can be found in prime gaps.
Patterns are easy to find. The key question is whether any pattern survives a
fair arithmetic null model and a prequential out-of-sample code-length test.

## 12. Paper Claim Discipline

The current paper can safely claim:

1. a reproducible prequential residual-coding protocol for prime gaps;
2. a successful positive control against the weak \(B_0\) baseline;
3. no detected positive residual predictive information beyond \(B_1(11)\) up
   to \(X=2^{26}\) under the tested residual predictors;
4. a finite-scale negative result that supports the interpretation that the
   detected \(B_0\) signal is wheel/arithmetic structure.

The current paper should not claim:

1. that prime gaps contain no residual information at all;
2. that all possible \(B_2\), \(B_3\), or stronger arithmetic baselines have
   been exhausted;
3. that the finite-scale result proves an asymptotic theorem;
4. that the implemented \(B_2\)-style prototypes are canonical tuple-corrected
   null models.

This claim discipline is the safest route to a credible paper.

## 13. Relation to Existing Work

This program combines several existing lines of work but asks a different
primary question.

Random models for primes, beginning with Cramer-type models [1] and refined by
Granville-style and random-sieve models [2], provide the conceptual basis for
\(B_0\) and \(B_1\). Modern random-sieve models explicitly encode the idea that
small-prime divisibility constraints should be built into the null model rather
than rediscovered as residual structure.

Hardy-Littlewood prime tuple heuristics and the singular series provide the
mathematical basis for \(B_2\). Biases in consecutive primes modulo \(q\), such
as those studied by Lemke Oliver and Soundararajan [3], show that consecutive
primes have real local arithmetic dependence, but also that much of this
dependence is explainable by tuple heuristics.

Empirical studies of entropy, correlation, periodicity, and scaling in prime
gaps [4] are relevant but usually do not impose a hierarchical arithmetic null
model together with penalized prequential prediction. Conversely, prequential
analysis [5], minimum description length [6], universal coding [7], and
context-tree methods [8] are standard in statistics and information theory, but
are not usually used as the organizing invariant for residual structure in
prime gaps.

The proposed contribution is therefore the synthesis:

> residual sequential information in prime gaps, measured as penalized
> prequential code gain over an explicit arithmetic null hierarchy.

## 14. Main Open Problems

### Problem 1: Canonical Baselines

Construct a nested family \(B_k\) that is strong enough to incorporate known
arithmetic structure but weak enough not to reconstruct the prime sequence in
the test window.

### Problem 2: Existence of \(\rho_k\)

For fixed \(k\), determine whether

\[
\rho_k=\lim_{X\to\infty}R_k(X)
\]

exists for natural universal residual codes \(U_k\).

### Problem 3: Vanishing Under Strong Baselines

Determine whether

\[
\rho_\infty=0
\]

under a natural arithmetic hierarchy. This is the expected outcome under the
view that Hardy-Littlewood/random-sieve structure exhausts sequential
predictability.

### Problem 4: Positive Residual Information

Determine whether there exists any admissible hierarchy for which

\[
\rho_\infty>0.
\]

Such a result would indicate a form of sequential structure in prime gaps not
captured by standard local arithmetic corrections.

### Problem 5: Finite-Sample Detection

Develop finite-sample significance tests for \(R_k(X)\) using synthetic
baselines, block resampling, and regret bounds for universal online codes.

## 15. Summary

The proposed invariant is not absolute entropy, compressibility, curvature, or
randomness. It is the residual prequential code gain

\[
R_k(X)=
\frac{L_{B_k}(G_X)-L_{U_k}(G_X)}{|I_X|}.
\]

This measures how many bits per gap are saved by an admissible universal
sequence predictor after a specified arithmetic null model has already been
given local density, wheel structure, finite tuple corrections, and possibly
controlled global analytic information.

If \(R_k(X)\) vanishes as the baseline is strengthened, the conclusion is not
failure but a meaningful negative result: the known arithmetic structure has
exhausted the detectable sequential information at the tested scale and model
resolution. If a positive residual persists after all corrections and
penalties, the result would point to a genuinely new form of predictive
structure in the gaps between consecutive primes.

The completed paper-scale suite supports the former interpretation for the
tested \(B_1(11)\) setting through \(X=2^{26}\).
